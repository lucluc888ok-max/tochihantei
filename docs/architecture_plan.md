# 用地仕入れ収益シミュレーター プロジェクト設計書

本ドキュメントは、不動産デベロッパー向け「用地仕入れ収益シミュレーター」のシステム基本設計、技術スタック、およびディレクトリ構造を定義します。

## 1. システム要件と技術選定

### フロントエンド (UI/UX担当)
- **フレームワーク**: React (Vite)
- **スタイリング**: Tailwind CSS
- **コンポーネントライブラリ**: shadcn/ui または MUI (モダンで素早いUI構築のため)
- **データフェッチ**: React Query
- **図面・区画割り描画**: Konva.js または Fabric.js (Canvas上で区画を操作・描画するため)
- **帳票出力 (PDF化)**: react-pdf またはブラウザの標準印刷機能

### バックエンド (コアロジック・API担当)
- **フレームワーク**: Python - FastAPI (高速なレスポンス、型安全、非同期処理に優れているため)
- **PDF解析 (物件概要書抽出)**:
  - **テキストベースのPDF**: `pdfplumber` または `PyMuPDF`
  - **AI情報抽出**: Gemini API または OpenAI API (非定型フォーマットから「地番、面積、用途地域、制限、権利関係」を高い精度で抽出するため)
- **外部API連携**: `httpx` (不動産情報ライブラリAPIへの非同期通信)
- **データ処理・収益計算**: `pandas`, `numpy`
- **地理・区画計算処理**: `shapely`, `geopandas` (面積計算や区画割りの幾何学的な処理用)

## 2. ディレクトリ構造

## 3. 実装の進め方

1. **フェーズ1 (基盤構築 & PDF解析機能)**
   - `frontend` に Vite+React+Tailwind の環境を構築。
   - `backend` に FastAPI の環境を構築し、PDFアップロードエンドポイントを作成。
   - `pdfplumber` とプロンプトを用いて、PDFから「地番、面積、用途地域など」を抽出するAPI処理を完成させます。

2. **フェーズ2 (外部API連携 & 収支計算機能)**
   - 抽出した情報（地番や用途地域等）を元に、「不動産情報ライブラリAPI」へ通信し周辺の成約事例を取得する機能を実装。
   - バックエンドに「仕入・建築・販売・利益」を自動算出する計算ロジック（`pandas`利用）を組み込みます。

3. **フェーズ3 (区画割りとUI統合)**
   - フロントエンドから情報を入力・確認し、結果をテーブルやグラフで表示するUIを実装。
   - キャンバス(`Konva.js`)を用いた区画割り案の自動生成モデルおよび手動調整UIと、最終的な収支表出力を統合します。
