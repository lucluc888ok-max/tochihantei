# マネタイズ要件・環境・費用まとめ

作成日: 2026-04-26

---

## 1. 背景・方針

- **現状**: ログイン制限なし・誰でも無制限に利用可能
- **目的**: 将来的な有料化に向けた利用者制限・課金基盤の整備
- **競合**: つくるAI（デベNAVI）月10万円/3名 → 法人高単価モデル
- **差別化**: 低コスト・シンプル・個人〜小規模チーム向け

---

## 2. プラン設計（案）

| プラン | 想定価格 | 対象 | 制限 |
|--------|---------|------|------|
| 無料（トライアル） | 0円 | 初回検討者 | シミュレーション月5回・履歴保存なし |
| 個人プラン | 月3,000〜5,000円 | 個人投資家・仲介 | 無制限・全機能 |
| チームプラン | 月15,000〜30,000円 | 小規模デベロッパー | 複数名・物件共有 |

> 価格・制限値は市場反応を見て変更前提

---

## 3. 必要機能要件

### 3-1. 認証
- メールアドレス＋パスワードでの登録・ログイン
- Googleアカウントログイン（オプション）
- パスワードリセット（メール送信）
- セッション管理（ログイン状態の保持）
- ログインなしはアクセス不可（または試用回数のみ可）

### 3-2. 利用制限
- シミュレーション実行回数の管理（月次リセット）
- プランごとの機能解放（例: CSV出力・天空率は有料のみ）
- 上限到達時のアップグレード誘導UI

### 3-3. 決済
- クレジットカードによる月額サブスクリプション
- 領収書の自動発行
- プラン変更・解約のセルフサービス対応

### 3-4. ユーザー管理（管理者）
- ユーザー一覧・プラン確認
- 手動でのプラン変更（初期フェーズは管理者操作でOK）
- 利用状況の把握

### 3-5. メール通知
- 登録確認メール
- パスワードリセットメール
- （将来）請求・更新通知

## 4. 推奨技術スタック

| 役割 | 採用サービス | 理由 |
|------|------------|------|
| 認証 | Firebase Authentication | 無料・Googleログイン対応・実績多数 |
| データベース | Supabase | PostgreSQL・無料枠あり・Railwayとの相性良好 |
| 決済 | Stripe | サブスクリプション管理が充実・国内実績多数 |
| メール送信 | Resend | 無料枠3,000通/月・シンプルなAPI |

## 6. マネタイズ後の追加環境・費用

### 追加サービス

| サービス | 無料枠 | 有料プラン | 備考 |
|---------|--------|-----------|------|
| Firebase Authentication | 月10万認証まで無料 | 従量課金（実質無料） | |
| Supabase | 500MB・月50万APIリクエスト | $25/月〜 | 規模拡大時に移行 |
| Stripe | 固定費なし | 売上の3.6%（手数料のみ） | 5,000円プランなら180円/件 |
| Resend | 月3,000通無料 | $20/月〜（月5万通） | 当面は無料枠で十分 |

### フェーズ別費用試算

| フェーズ | 状況 | 月額費用（目安） |
|---------|------|----------------|
| 現状 | 認証なし | 約750円 |
| ローンチ直後（〜10名） | 全サービス無料枠内 | 約750円（変わらず） |
| 成長期（10〜100名） | Supabase有料移行の可能性 | 約4,500円 |
| 拡大期（100名〜） | Railway増強・Supabase拡張 | 1〜2万円 |

> **Stripe手数料は売上発生後のみ。固定費の増加は最小限に抑えられる。**

## 7. 実装優先順位

| 順序 | 要件 | 難易度 | 備考 |
|------|------|--------|------|
| 1 | ログイン認証 | 中 | Firebase Authで簡略化可 |
| 2 | 利用回数制限 | 低〜中 | Supabaseでカウント管理 |
| 3 | Stripe決済連携 | 中 | Webhookの設定が必要 |
| 4 | プラン管理 | 中 | DBでユーザー×プランを管理 |
| 5 | 管理画面 | 中 | 初期は手動対応でも代替可 |
| 6 | チーム機能 | 高 | 後回しでOK |

---

## 8. 実装時の参考

```
フロントエンド (Vercel)
  └── Firebase Auth SDK（認証状態管理）
  └── Stripe.js（決済フォーム）

バックエンド (Railway / FastAPI)
  └── Firebase Admin SDK（トークン検証）
  └── Supabase Client（ユーザー・利用回数管理）
  └── Stripe API（サブスクリプション管理）
  └── Resend API（メール送信）
```
