# 用地仕入れシミュレーター 精度向上・実用化に向けたロードマップ

現在のプロトタイプ（UIと基本推論エンジンの疎通確認）から、実務の過酷な精度要求に応えうる「**本番レベルの完全自動採算チェッカー**」へ昇華させるための4つの高度化フェーズ計画です。

---

## 🏗️ 1. PDF抽出精度の劇的な向上（AI・OCRの高度化）
現在：正規表現によるパターンマッチング（モックロジック）
**課題**：仲介業者ごとに物件概要書のフォーマットがバラバラで、手書きやFAX・低画質PDFなど表現のゆらぎが激しいため、単純なプログラムではエラーになる。

**【改善アプローチ】**
1. **マルチモーダルLLM（Gemini 1.5 Pro）の導入**
   テキストベースの抽出を廃止し、PDF自体またはPDFを画像化したものを直接LLMに読み込ませます。
2. **Structured Outputs (JSON化) による一発構造化**
   「地番」「用途地域」「面積（全体・私道負担）」「権利形態（底地・借地）」「特記事項」を、どんな崩れたフォーマットからでも高精度でJSONデータとして確実に抽出させます。
3. **ヒューマン・イン・ザ・ループ (修正UIの提供)**
   AIが『少し自信がない箇所』や抽出した根拠となるPDF上の文字をUIでハイライトし、人間が1秒で確認して手動修正できるエリアを設けます。

---

## 📊 2. 相場データの精度向上（リアルタイム取引データの網羅）
現在：ダミーCSVによる静的な周辺相場の平均計算
**課題**：現実の相場は「駅からの距離」「整形地か不整形地か」「前面道路の幅員」によって坪単価が大きく変動する。

**【改善アプローチ】**
1. **国土交通省「不動産情報ライブラリAPI」との本番連動**
   該当住所から常に「半径1km以内の直近2年間の類似物件事例」をWeb APIからクローリングし、最新の相場を取得します。
2. **AIペナルティ・アルゴリズムの適用**
   取得した相場の平均値に対し、PDFから読み取った「最寄り駅からの徒歩分数」「間口が狭い旗竿地か」「不整形か」などのマイナス要素を自動加味して、その土地限定の『精密な適正相場単価（上限買値）』を算出します。

---

## 📏 3. 建築・法規制ロジックの精度向上（自動面積・ボリューム計算）
現在：単純な「2項道路は中心から2m一律にセットバック」という簡易概算
**課題**：実際の「有効宅地面積（建物を建てられる広さ）」は建蔽率の特例や、間口寸法によって細かく変わってくる。

**【改善アプローチ】**
1. **緩和条件の自動判定**
   特記事項等の文字から「角地」や「防火地域内の耐火建築物」を拾い上げ、自動的に建蔽率に `+10%` または `+20%` の緩和ボーナスを加算するロジックを実装します。
2. **より正確な後退面積・私道計算**
   図面から概算の「間口寸法」を推測させることで、正確なセットバック面積を算出し、建蔽率・容積率上限での「最大建築可能な延床面積（ボリューム）」をシミュレーションします。

---

## 💰 4. 事業収支（デベロッパー原価計算）の精緻化
現在：「売上 - 仕入 - どんぶり勘定の建築費 = 純利益」という簡易式
**課題**：実際の不動産開発では、税金や金利などの「見えない経費」が採算を大きく狂わせる。

**【改善アプローチ】**
以下の項目を内部計算ロジックに自動的に組み込み、粗利から引いていきます。
1. **仲介手数料**: （売買価格 × 3% + 6万）× 消費税 (仕入・販売の両方)
2. **既存建物の解体費**: PDF情報から「更地渡し」か「古家付き」かを判別し、古家付きなら木造坪5万円などの解体費を自動見積もり加算。
3. **諸経費・税金**: 登録免許税、不動産取得税、固都税精算金、印紙代を自動算出。
4. **金融機関借入金利**: 開発から販売までの期間（例：10ヶ月）のプロジェクトファイナンス金利を原価に上乗せ。
